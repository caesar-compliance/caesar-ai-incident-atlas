# T059 Decisions

**Task:** Hosted Activation Preflight + Local Supabase Migration Smoke + Operator Go-Live Checklist  
**Date:** 21 May 2026

---

## DEC-132 — Local Migration Smoke Checker Mode

**Decision:** The smoke checker must skip gracefully if local Supabase CLI/Docker is unavailable, returning `pass: true` with mode `skipped_local_runtime_missing`.

**Rationale:** T059 is a preflight preparation task, not a local infrastructure setup task. We should not fail validation just because the operator hasn't installed local Supabase tooling yet.

---

## DEC-133 — Preflight Script Non-Destructive Design

**Decision:** `preflight-hosted-activation.mjs` is a readiness checker only. It must never attempt remote connections, never apply migrations, never deploy Workers.

**Rationale:** The script generates a checklist for the human operator. The actual activation steps are documented but require manual execution with CT approval.

---

## DEC-134 — Activation Manifest Auto-Generation

**Decision:** `hosted-activation-manifest.json` is generated by the preflight script, not hand-maintained.

**Rationale:** Keeps manifest synchronized with actual repository state. Prevents stale documentation.

---

## DEC-135 — Command Checklist Deterministic Output

**Decision:** `print-hosted-activation-commands.mjs` must produce deterministic output (no timestamps, no randomness, no env-dependent content).

**Rationale:** Allows the checklist to be compared across runs and committed to documentation safely.

---

## DEC-136 — Safety Validator Coverage Expansion

**Decision:** Update `validate-hosted-sync-safety.mjs` to also validate T059-generated files are sanitized.

**Checks added:**
- `local-migration-smoke.json` contains no secret values
- `hosted-activation-preflight.json` contains no secret values
- `hosted-activation-manifest.json` contains no secret values
- No real `SUPABASE_URL` appears in generated files
- No activation commands were executed (files show pending state)

**Rationale:** Defense in depth. Generated files could theoretically be misconfigured; safety validator must catch this.
